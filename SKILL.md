# APPSTV-CRM Database Schema Reference

**Database:** `sohailshaikhz_appstv_crm`
**Server:** MySQL 8.0.42 | **PHP:** 8.3.30 | **Environment:** XAMPP (localhost)

---

## Coding Conventions

- Every table has an auto-increment `id` (int) as the primary index key — not used in PHP class logic
- Business primary key is a varchar(50) unique ID (e.g. `customer_id`, `order_id`, `psnw_id`)
- Unique IDs are 10-char strings generated by `generateUniqueID()` in `library/functions.php` — **never redeclare this**
- ID format: `dd` + 4-digit milliseconds + 4 random digits (e.g. `dd12345678`)
- Timestamps stored as `varchar(100)` in ISO 8601 format
- PHP classes live in `plugins/{plugin_dir}/includes/{filename}.php`
- CRUD method names mirror the entity (e.g. `createWarranty()`, `readInvoice()`)
- Properties are private; constants named `DB_FIELD_*`
- Constructor validates uniqueness of generated ID against DB before proceeding
